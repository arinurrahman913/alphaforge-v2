## 2. Repo Kode (`alphaforge-core-v2`)

### Sudah dikerjakan & di-push
- **Implementasi awal lengkap**: Layer 1 (12/12 komponen), Layer 2 end-to-end (Screening → Evidence → Knowledge → Confidence+Peer → Risk/Red-Flag → 3 modul reasoning → Aggregator), CLI, 18 unit test (logic murni, tanpa network).
- **Fix hasil review kode:**
  - `screen_universe()` sekarang pakai batching + delay (`YF_BATCH_SIZE`/`YF_BATCH_DELAY_SECONDS`) — sebelumnya rawan kena rate-limit Yahoo Finance.
  - `compute_confidence()` sekarang menghitung kebaruan data (`evidence_age_days`) — sebelumnya cuma pakai completeness + source count + flags.
  - `peer_comparison.py` menandai `low_sample_size` kalau peer group < 3 saham.
  - `pipeline.py` mencatat peer yang gagal ditarik datanya secara eksplisit (`peer_failures`) — sebelumnya silent-skip.
  - Cleanup angka ambang mega-cap yang formatnya membingungkan.
- **`report_formatter.py`** — output CLI defaultnya sekarang ringkasan enak dibaca (Confidence → Risk → 3 Views → Peer), bukan JSON mentah. JSON tetap tersedia lewat `--json`.
- **Tervalidasi dengan data live** (ticker AAPL, peer MSFT/GOOGL) — pipeline jalan end-to-end, red-flag check berhasil menangkap litigasi OpenAI-Apple yang sedang berjalan, tiga modul reasoning menghasilkan kesimpulan berbeda-beda sesuai lensanya masing-masing.

### Diketahui belum lengkap / masih kurang
- **`market_breadth` selalu `null`** — butuh daftar konstituen index (S&P 500/NASDAQ) yang belum disuplai ke `build_market_context_package()`. Bukan bug, memang belum diberi input.
- **`institutional_ownership_detail` selalu `missing`** — Finnhub endpoint premium-only (403), bukan gratis seperti asumsi awal spec. `institutional_ownership_pct` (persentase saja) tetap ada dari Yahoo Finance.
- **`screen_universe()` belum pernah dites ke ribuan ticker riil** — cuma dites logic-nya, belum divalidasi jalan lancar untuk full market NASDAQ+NYSE.
- **Historical Tracking / Decision Journal belum diimplementasikan** — diakui eksplisit sebagai kandidat v2.1 di spec.
- **Bobot & kriteria 3 modul reasoning masih interpretasi awal**, belum dikalibrasi/divalidasi — spec sendiri menyatakan ini seharusnya didiskusikan terpisah sebelum implementasi final.
- **Peer discovery belum otomatis** — `peer_tickers` masih harus disuplai manual oleh caller, sistem belum bisa cari peer sejenis sektor/industri sendiri.
- **Respons red-flag di 3 modul masih level umum** (moderate/elevated), belum spesifik per jenis flag (misal flag akuntansi harusnya lebih ditonjolkan di Quality/Compound dibanding di Speculative).
